# 飞呀飞 — 社区二手物品交易平台开发计划

## 项目总览

基于 React + Vite + Tailwind + Supabase (Lovable Cloud) 构建的全功能社区二手交易平台，聚焦家长互助、学生物品循环利用。

### 1. 静态资源与主题

## 阶段二：用户体系

### 4. 注册页面

- 按需求顺序排列7个字段，均为必填项：账号(手机号，只允许填写数字)、密码（长度≥8位）、称呼、所在城市(三级联动选择器，数据来自ChinaCitys.json)、小区(含定位按钮)、孩子就读学校(含定位按钮)、年级+学期：
  - 账号：必填项，必须为纯数字，提示语“请填写真实手机号，便于通过手机号联系完成交易”。
    登录密码：必填项，长度≥8位。
    称呼：必填项，示例“小明妈妈”，提示语“该称呼将显示在物品列表和详情页中”。
    所在城市：必填项，需提供真实中国大陆省、市、区/县行政区划数据供用户选择，提示语“选择正确区域有助于附近的家长精确查找到您发布的闲置物品”。省、市、区/县需要完整，比如重庆市可以通过市辖区选择秀山县。
    小区名称：选填项，提示语“请填写完整的小区名称，建议以地图App上的名称为准，方便附近家长精准查找到您发布的闲置物品”。
    孩子就读学校：选填项，提示语“请填写完整的学校名称，建议以地图App上的名称为准，方便附近家长精准查找到您发布的闲置物品”。
    孩子的年级和学期：选填项，年级选项涵盖幼儿园、小学一年级至高一所有年级，学期选项为“上学期”“下学期”（年级为幼儿园时不可选择学期），提示语“选择孩子当前的年级和学期，系统将自动优先展示匹配的物品。小初高在每年3月1日和9月1日自动升级学期”。
- 高德地图定位弹窗：使用 JS API 2.0，Key 和安全密钥已提供，拖拽标记自动解析小区/学校名称，为了安全起见，将API密钥作为环境变量进行存储，要求：
  - 交互逻辑： - 网页上有一个「定位」按钮，用户点击后唤起有高德地图的弹窗； - 自动触发高精度定位，地图聚焦到用户当前位置并显示标记； - 支持用户拖动地图上的标记调整位置，松开后重新解析位置信息。
  - 核心功能： - 仅提取并显示「小区（住宅区）、学校等」的标准名称（只显示小区名，如“万科城”，不要省市区街道等冗余信息）； - 若定位/拖动后未识别到小区，显示“未识别到小区名称”； 。
  - 技术要求： - 使用高德地图2.0版本JS API； - 代码完整可运行， - 处理定位失败、接口调用失败等异常情况； - 样式简洁，结果显示区域清晰。
  - 非常重要：用户拖拽标记松开后自动重新获取新位置的小区和学校名称（仅显示纯小区和学校名称）。
  - 高德地图JS API的Key：e736815f668a6d3156ff658f6961889c；安全密钥：c8ce16fdbcc36b81106a0aab89f90cae
- 用户协议弹窗（自动生成协议文本）
- 注册成功自动登录并返回进入注册/登录页面的前一页

## 阶段三：首页与核心功能

### 6. 首页

- 顶部提示条（交易风险提示）
- Logo + 平台名称 + 副标题，点击返回首页
- 城市筛选（默认用户城市，支持搜索并选择城市进行切换）
- 搜索栏（关键词搜索 + 取消搜索）
- 筛选区：小区、学校（下拉搜索多选）、类型（书籍/其他）、年级、学期、成色（含悬停说明）、重置按钮
- 登录用户筛选默认匹配个人信息，修改后保留状态直到重新登录
- 商品卡片列表：封面(1:1.42)、名称、作者(书籍)、学校、年级学期、成色、发布者称呼+店铺按钮、价格、脱敏/完整手机号、交易中标记、加入购物车按钮
- 按发布时间倒序排列
- 无物品时显示提示+发布按钮

### 7. 商品详情页

- 完整信息展示 + 成色说明
- "加入购物车"按钮（飞向购物车动画）
- "我也要卖此书"按钮（复制信息到发布页）
- 未登录查看手机号跳转登录
- 返回上一页

### 8. 发布页面

- 书籍/其他两个 Tab
- 书籍：封面*、书名*、作者、译者、出版社、出版时间、适用年级(多选不可选学期)、书籍标签*、成色*、售价*
- 其他：图片、名称*(≤10汉字)、适用学校、适用年级、物品描述、瑕疵描述、售价*
- 自动填充用户信息中已有的字段
- 发布成功跳转首页

### 9. 购物车

- 按卖家分组，显示卖家信息
- 物品：封面(可点进详情)、名称、成色/描述、售价、移除按钮
- 一键全选某卖家商品
- 确认购买弹窗（显示完整提示信息和买家信息）

### 10. 订单页面

- 三个 Tab：全部、我买到的、我卖出的
- 订单卡片：物品信息、交易状态、时间、双方信息
- 确认收货（主色突出按钮）→ 触发评价
- 取消交易 → 物品恢复上架
- 评价功能：买家评配合度+描述一致性，卖家评配合度，五星制

### 11. 个人主页/店铺

- 用户信息 + 综合评分
- 在售/交易中物品列表
- 评价展示（评价者脱敏）
- 复制主页链接（含推荐文案前缀）

### 12. 我的发布

- 所有物品列表（含状态）
- 修改、下架、已售出、删除操作

### 13. 个人中心

- 信息展示与修改（手机号不可改）
- 退出登录

## 阶段五：管理后台

### 15. 管理后台 `/tianadmin`

- 仅 admin/moderator 角色可访问
- **数据统计**：用户总数、书籍/非书籍数量、已售出/下架数量、浏览量；趋势图（用户增长、上架、售出、下架、浏览）；书籍标签饼图；评价柱状图；价格区间柱状图
- **用户管理**：用户列表、重置密码、登录日志、禁用/启用
- **物品管理**：物品表格、强制下架、状态筛选、名称搜索
- **评价管理**：所有评价列表、按评分排序、按类型筛选、删除评价（同步更新综合评分）

## 技术要点

- **响应式**：Tailwind 断点适配 PC/移动端，底部 Tab 栏 (mobile) vs 顶部导航 (desktop)
- **图片上传**：Supabase Storage，封面图压缩处理
- **高德地图**：通过 script 标签动态加载 JS API 2.0，安全密钥通过代理或 `_AMapSecurityConfig` 配置
- **行政区划**：ChinaCitys.json 放 `public/data/`，按需 fetch 加载，三级联动选择器
- **状态管理**：React Context 管理用户登录态和筛选状态
- **图表**：使用已安装的 recharts 库
- **自动任务**：Supabase pg_cron 或 Edge Function 定时执行学期升级和默认评价

## 实施顺序

由于项目规模较大，建议分批实施：

1. **第一批**：基础设施（数据库、主题、路由、布局组件、导航）+ 注册/登录
2. **第二批**：首页（筛选、搜索、商品列表）+ 商品详情
3. **第三批**：发布页面 + 购物车 + 订单
4. **第四批**：个人主页/店铺 + 我的发布 + 个人中心
5. **第五批**：管理后台
6. **第六批**：高德地图定位、学期自动升级、自动评价等进阶功能
